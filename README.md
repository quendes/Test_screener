+# Binance Futures 1m Screener (проторговка/запил)
+
+Скрипт сканирует фьючерсы Binance (USDT perpetual, 1m), ищет графики с трендом и мелкими откатами (похожими на ваши примеры), рисует свечной скриншот и отправляет его в Telegram-бота.
+
+## Что делает
+
+- Берёт топ-N USDT perpetual контрактов по объёму.
+- Для каждого инструмента загружает 1m свечи.
+- Считает эвристический `score` паттерна:
+  - направленный тренд,
+  - стабильность (R² линейной регрессии),
+  - ограниченная глубина откатов,
+  - «запиленность» (локальные экстремумы),
+  - адекватная волатильность.
+- При попадании под пороги отправляет в Telegram:
+  - скриншот графика,
+  - метрики сигнала.
+- Защищается от дублей (не шлёт один и тот же символ в пределах той же 1m свечи).
+
+## Быстрый старт
+
+```bash
+python -m venv .venv
+source .venv/bin/activate
+pip install -r requirements.txt
+cp config.example.env .env
+# заполните TELEGRAM_BOT_TOKEN и TELEGRAM_CHAT_ID
+python screener.py
+```
+
+## Настройки `.env`
+
+- `TOP_SYMBOLS` — сколько пар сканировать (по умолчанию 50).
+- `CANDLES_LIMIT` — сколько свечей грузить (220).
+- `LOOKBACK_BARS` — окно анализа (120).
+- `SCAN_INTERVAL_SEC` — интервал сканирования в секундах (60).
+- `SCORE_THRESHOLD` — общий порог качества паттерна (0.62).
+- `MIN_TREND_PCT` — минимальный рост за окно анализа, % (0.8).
+- `MAX_PULLBACK_PCT` — максимум просадки внутри окна, % (1.0).
+- `STATE_FILE` — файл антидубликатов.
+
+## Важные замечания
+
+- Это **эвристический** поиск, а не гарант сигнала.
+- Пороговые значения нужно калибровать под ваш стиль и волатильность рынка.
+- Для production лучше запускать под `systemd`/`supervisor` и добавить retry/backoff на сеть.
